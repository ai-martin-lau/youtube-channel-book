# 小书结构模板 (BOOK.md)

按主题组织，不按视频组织。建议结构：

```markdown
# 《<频道名> 精华：<一句话副标题>》
> 基于该频道 N 个视频字幕蒸馏 · <日期>

## 前言：这本书讲什么
- 作者是谁、频道在讲什么、为谁而写
- 一段话讲清作者最核心的世界观(他所有视频的总纲)

## 全书地图(一页速览)
- 用 5–10 条要点列出全书骨架，让读者 30 秒抓住全貌

## 第 1 章：<主题>
### 核心论点
### 框架 / 方法论(带步骤)
### 怎么落地(可执行清单)
### 金句
### 出自(视频标题)

## 第 2 章 … 第 N 章
(同上结构，每章一个大主题)

## 行动手册(把全书变成可执行清单)
- [ ] 第一周做什么
- [ ] 持续做什么
- [ ] 避免什么

## 速查卡(一页)
- 核心公式 / 框架名 / 关键数字 / 高频金句，浓缩成一页便于反复看
```

## 章节怎么定

不要预设章节。先把所有子 Agent 的「核心主题」聚类，自然涌现 4–8 个大主题，
每个主题成一章。常见聚类维度(因频道而异)：心态/世界观、定位/选题、
具体打法/SOP、工具与系统、变现、成长复利、常见误区。

## 质量自检

- [ ] 一个没看过该频道的人，读完能复述作者的核心方法吗？
- [ ] 有没有作者**特有**的框架和术语，而不是放之四海皆准的废话？
- [ ] 关键处有没有**具体数字、步骤、案例**？
- [ ] 高频反复出现的观点有没有被当作主线突出？
- [ ] 金句是否保留了原话的力量？
