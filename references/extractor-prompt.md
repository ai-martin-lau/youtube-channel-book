# 子 Agent 提炼模板

发给每个并行子 Agent 的 prompt 用下面结构填充(替换 `<...>`)：

---

你在帮我把一个 YouTube 频道蒸馏成一本学习小书。下面是其中一批视频的英文字幕(已转纯文本)。

**字幕目录**：`<txt 目录绝对路径>`
**你负责的视频**(逐个用 Read 工具读 `<目录>/<videoId>.txt`)：
```
<videoId>  <标题>
<videoId>  <标题>
... (本批 12–18 个)
```

逐个读完后，输出**这一批的结构化精华**(不要复述原文、不要逐视频流水账，要跨视频归纳)：

1. **覆盖的核心主题**(3–8 个，每个一句话)
2. **可复用的框架 / 心智模型 / 方法论**：作者反复用的那些。给出名字 + 怎么用 + 步骤。
3. **具体可执行的做法**：带数字、节奏、清单、工具、模板的那种(越具体越好)。
4. **反直觉 / 反主流的观点**：作者跟大众说法不一样的地方。
5. **金句**(5–15 条)：原话(英文)，每条标注出自哪个视频标题。
6. **作者独有的术语 / 概念**：他自创或爱用的词，附解释。

要求：
- 只提炼**这个作者真正说的、特有的**东西，不要填充通用大道理。
- 同一观点在多个视频反复出现 → 标注「高频(出现在 N 个视频)」，这是作者的核心信念。
- 字幕是自动生成的，可能有错别字/断句乱，按语义理解。
- 直接返回结构化精华文本即可(这是数据，不是给人看的汇报，不要寒暄)。
